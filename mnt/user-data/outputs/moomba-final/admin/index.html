Performance checklist for your developer or hosting team. These optimizations directly improve Lighthouse scores and mobile load times — critical for tourism visitors on mobile data.

Target Lighthouse
90+
Mobile performance score
Target Load Time
<2s
On 3G mobile connection
Images & Assets
✓
Native lazy loadingAll images use loading="lazy" + explicit width/height to prevent CLS. Already implemented in this build.
!
Image compressionRun all uploaded photos through Squoosh.app or TinyPNG before uploading. Target: <200KB per image, use WebP format where possible.
!
CDN for static assetsServe CSS/JS/images via Cloudflare CDN. Add "Cache-Control: max-age=31536000, immutable" for hashed asset filenames.
✓
Responsive imagesAll images have explicit width/height attributes. Add srcset with 2x versions for hero images when deploying.
Fonts & CSS
✓
Font display: swapGoogle Fonts loaded with display=swap — text shows immediately in system font while custom fonts load.
✓
Preconnect to font originsfonts.googleapis.com and fonts.gstatic.com preconnected in <head>. Eliminates DNS lookup delay.
!
Critical CSS inliningFor production: inline above-the-fold CSS (nav, hero) into a <style> tag per page, then load the rest async. Eliminates render-blocking CSS.
!
CSS/JS minificationRun style.css and main.js through a minifier (eg. esbuild, Vite, or online tools). Reduces file size by ~40%.
Server & Hosting
!
Gzip / Brotli compressionEnable in your server config (Nginx: gzip on; brotli on;). Reduces HTML/CSS/JS transfer size by 60–80%.
!
HTTP/2 or HTTP/3Enable on your hosting platform. Allows multiplexed asset loading — critical on mobile where TCP connections are expensive.
✗
Preload hero imageAdd <link rel="preload" as="image" href="hero-image.jpg"> in <head> on index.html to improve LCP score.
!
Service Worker (offline)Add a simple service worker to cache the site shell. Tourists often have spotty mobile data — this keeps the site usable.
SEO & Discoverability
✓
Schema.org structured dataBarOrPub JSON-LD schema on homepage with hours, geo-coordinates, cuisine, and social links.
✓
Open Graph & Twitter cardsAll pages have OG meta tags for rich social sharing previews.
✓
Semantic HTML & ARIACorrect landmark roles, aria-labels, and heading hierarchy throughout.
!
Google Search ConsoleSubmit sitemap.xml to Google Search Console. Monitor Core Web Vitals reports monthly.
!
Sitemap.xmlGenerate and submit a sitemap listing all pages. Use a free tool like xml-sitemaps.com.
